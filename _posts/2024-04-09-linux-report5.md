# 프로그램 설치 명령어 dpkg
## 프로그램 설치 명령어 dpkg
**dpkg(debian package)의 개요**
- 우분투에서 패키지(프로그램)을 설치할 때 가장 많이 사용되는 명령어는 **dpkg**와 **apt-get(advanced packaging tool)**이다.
- apt-get이 나온 뒤로는 dpkg의 기능이 포함되어 있기 때문에 apt-get을 사용.
- apt-get은 별개의 명령어가 아닌 dpkg를 포함한 확장한 개념에 가깝기에 dpkg의 개념의 습득을 요함.

- 데비안 리눅스가 초창기 리눅스의 문제였던 새로운 프로그램 설치의 어려움을 해결하기 위해 윈도우의 설치 파일과 비슷한 원리인 프로그램 설치 후 바로 실행이 가능한 ‘패키지’라는 설치 파일을 제작했다.
- 이 패키지의 확장명은 *.deb이다.
<div class = "notice--success">
dpkg 개념의 습득을 요하는 이유는 apt-get이 의존성 문제나 패키지 설치 실패 등의 문제가 발생했을 때에 dpkg를 사용하여 문제의 원인을 분석하고 해결할 수 있기 때문
그 외로는 패키지 관리의 깊은 이해와 개인화, 커스터마이징이 가능하다는 점이 있음.
</div>
<br>

## 파일의 의미
우분투 사이트 저장소(repository)에는 많은 deb 파일이 있다.
그중에서 X 윈도우용 계산기 프로그램인 galculator을 예시로 가져왔다. deb 파일의 형식은 패키지에 따라 다를 수 있으나 일반적으로 ‘패키지명_버전-개정번호_아키텍처.deb’이다.

![1](/images/2024-04-09-linux-report5/Picture1.png)
<br>위의 galculator의 deb 파일 형식을 자세히 살펴보면
- **패키지명**: galculator – 패키지(프로그램)의 이름이다
- **버전**: 2.1.4 – 대개 세 자릿수로 구성된다. 주버전, 부버전, 패치 버전 순이다.
- **개정번호(revision number)**: 1 – 문제점을 개선할 때마다 붙이는 번호이며 높을 수록 좋다.
- **아키텍처**: amd64 – 64비트 CPU, 즉 이 파일을 설치할 수 있는 CPU를 말한다.
<div class = "notice--danger">
개정번호를 보면 높을 수록 좋다고 나와있지만 이는 경우에 따라 다르다. 개정번호는 새로운 기능과 개선사항, 버그 수정, 보안 업데이트, 지원 종료 예방 등으로 숫자가 높아지지만, 가끔은 새로운 버전이 이전 버전보다 안정성이 낮거나 새로운 문제를 발생시킬 수도 있다. 따라서 버전이 높다고 항상 좋은 것은 아니다. 
</div>

## dpkg의 단점
이전보다 획기적으로 편리해지기는 했으나 dpkg 명령어에도 단점이 있다.
그중 가장 큰 단점은 의존성으로, 우분투의 기본 웹 브라우저인 파이어폭스는 X 윈도우상에서만 가동되는데 X 윈도우가 설치되지 않은 상태에서는 파이어폭스를 설치할 수 없다.

이러한 문제를 해결한 것이 바로 apt-get 명령어이다.
apt-get은 dpkg의 단점을 개선하여 설치하려는 패키지가 의존하고 있는 모든 패키지를 한번에 설치해준다.
<br>

# 패키지 설치 명령어 apt-get
## apt-get의 개요
- apt-get 명령어는 dpkg명령어의 의존성 문제를 해결해주고 *.deb패키지를 설치하는데 필요한 도구이다.
- dpkg 명령어의 경우 *.deb 파일을 미리 다운로드한 후 설치해야하지만, apt-get 명령어의 경우 우분투가 제공하는 deb파일 저장소(repository)에서 자동으로 다운로드하여 설치한다.
<br>

## apt-get의 기본사용법
![3](/images/2024-04-09-linux-report5/Picture3.png)
<br>

## apt-cache
- apt-cache 명령어를 사용하면 패키지를 설치하기 전에 패키지에 대한 정보나 의존성 문제를 사전에 확인할 수 있다.
```
apt-cache show[패키지 명]
```
패키지의 정보를 보여준다.
```
apt-cache depends[패키지 명]
```
패키지의 의존성 정보를 보여준다.
```
apt-cache rdepends[패키지 명]
```
패키지에 의존하는 다른 패키지의 목록을 보여준다.
<br>

## apt-get의 작동방식과 설정 파일
- apt-get install 패키지명 명령을 입력했을 때
    - /etc/apt/ 디렉터리의 핵심파일인 sources.list를 확인하고,
    - 우분투 패키지 저장소에 설치와 관련된 패키지 목록을 요청하고,
    - 설치와 관련된 목록만 다운로드
    - 설치할 패키지와 관련된 패키지의 이름만 화면에 출력
    - ’y’를 입력하면 설치에 필요한 패키지파일을 요청
    - 설치할 패키지 파일을 다운로드 및 자동 설치

<div class = "notice--success">
이때 apt-get-y install [패키지명]을 사용하면
위의 작업이 한번에 실행된다.
</div>

# 응급 복구와 GRUB 부트로더
## 응급 복구의 개요
- 리눅스를 설치한 후 오랜만에 사용할 때 root 사용자의 비밀번호가 기억이 나지 않는 경우가 흔히 존재한다.
- 위의 경우에 중요한 파일이 들어 있다면 문제가 발생할 수 있다. 
- 기존의 비밀번호를 알아내는 방법은 없지만, 새로운 비밀번호로 변경할 수 있다.
- 순서는 다음과 같다.
    - 서버 초기화하기
    - 비밀번호 변경 준비하기
    - 비밀번호 변경하기이다.
<br>

## root 사용자의 비밀번호 분실 시 조치하기
- Server(B) 초기화하기
    - root 사용자의 비밀번호를 잊었을때는 텍스트 모드 환경의 Server을 사용하기 위해 Server을 처음 설치 상태로 초기화한다.
    - Vmware을 종료한 후 C:\Linux(백업)\Server(B) 폴더를 C:\Linux\ 폴더에 통째로 복사한다.
- 비밀번호 변경 준비하기
    - Server(B)를 부팅하자마자 바로 검은 화면에서 마우스를 클릭하고 ESC를 여러 번 누르면 GRUB 메뉴 화면이 나타난다. 첫 번째 메뉴인 ‘*Ubuntu’가 선택된 상태에서 E를 누른다.
    - 아래로 내려가 맨 아래의 ‘linux /boot/vmlinuz-5.15.0-69-generic…’ 앞에 커서를 놓는다. 그리고 End를 눌러서 커서를 맨 뒤로 이동한 후 한 칸을 띄고 ‘init=/bin/bash’를 입력한다.
    - 그리고 ctrl+X 또는 F10을 눌러서 부팅한다.
- 비밀번호 변경하기
    - 별도의 로그인 절차 없이 부팅이 되고 root@(none):/# 프롬프트가 나타날 것이다. whoami 명령을 입력하여 현재 로그인된 사용자가 root인지 확인한다.
    - root 사용자의 비밀번호를 변경하기 위해 passwd 명령을 입력하고 새로운 비밀번호를 여덟 자 이상으로 설정한다.
    - 만약 현재/파티션이 읽기 전용으로 마운트 되어 있다면 mount –o remount,rw / 명령을 입력하여 / 파티션을 읽기/쓰기 모드로 다시 마운트한다.
    - 마지막으로 VMware 메뉴에서 [player]-[Power]-[Restart Guest]를 선택하여 시스템을 강제로 재부팅한다. 이제 root 사용자는 새로운 비밀번호로 접속할 수 있게 된다.
<br>

## GRUB(GRand Unified Bootloader) 부트로더
 우분투에서 기본적으로 제공하는 GRUB 부트로더는 우분투를 부팅할 때 처음 나오는 선택 화면을 말한다.

GRUB의 특징은 다음과 같다.
- 부트 정보를 사용자가 임의로 변경하여 부팅할 수 있다. 이는 부팅 정보가 올바르지 않더라도 수정하여 부팅할 수 있게 해준다.
- 다른 운영체제와 멀티부팅을 할 수 있다.
- 대화형 설정을 제공(ex.Anaconda)하므로 커널의 경로와 파일 이름만 알면 부팅이 가능하다.
- 셸 스크립트를 지원하므로 조건식과 함수를 사용할 수 있다.
- 동적 모듈을 로딩할 수 있다. 동적 모듈은 /boot/grub/i386-pc/ 디렉터리에 mod 파일로 존재한다.
- 그래픽 부트 메뉴를 지원하며 부트 스플래시(boot splash) 성능이 개선되었다.
- ISO 이미지를 이용하여 바로 부팅할 수 있다.

<div class = "notice--danger">
<li>GRUB는 대통합 부트로더 라는 뜻</li>
<li>셸 스크립트는 셸이나 CLI에서 돌아가도록 작성된 스크립트입니다. 바로 전 스크린에 나오는 사진이 셸 스크립트입니다.</li>
<li>동적 모듈은 간단하게 프로그래밍 실행 중에 동적(dynamic)으로 로드되고 연결되는 소프트웨어 컴포넌트입니다. 이러한 모듈은 실행 파일이나 응용 프로그램의 일부로 컴파일 되는 것이 아니라, 프로그램 실행 중에 필요할 때만 로드됩니다.</li>
<li>부트 스플래시는 시스템 부팅 과정 중에 화면에 표시되는 그래픽이나 로고 등을 말합니다.</li>
<li>ISO 이미지는 국제 표준화 기구(International Organization for Standardization)가 지정한 디스크 이미지 형식 중 하나로, 디스크에 저장된 파일 및 디렉터리 구조를 하나의 이미지 파일로 포장한 것입니다.</li>
</div>
<br>

/etc/default/grub 파일의 설정은 아래와 같다.
```
1   GRUB_DEFAULT=0
2   GRUB_TIMEOUT_STYLE=hidden
3   GRUB_TIMEOUT=0
4   GRUB_DISTRUPTOR=`lsb_release –I –s 2> /dev/null || echo Debian`
5   GRUB_CMDLINE_LINUX_DEFAULT=“quiet splash”
6   GRUB_CMDLINE=LINUX=“ ”
```
- 1행: GRUB 목록중에서 0번째(첫 번째)가 기본으로 선택되게 한다는 의미이다.
- 2행: 3행의 시간 동안 화면에 GRUB 목록이 보이지 않게 한다.
- 3행: 처음 화면이 나오고 자동으로 부팅되는 시간을 초 단위로 설정한다. 0이면 기다리지 않고 부팅이 진행되고, -1이면 자동으로 넘어가지 않고 사용자가 직접 엔트리를 선택할 때까지 기다린다.
<br>

## GRUB 부트로더 변경 및 비밀번호 설정하기
- Server(B) 부팅 준비하기
    - GRUB 부트로더를 변경하고 비밀번호를 설정하기 위해 Server를 처음 설치 상태로 초기화한다.
    - Vmware을 종료한 후 C:\Linux(백업)\Server(B) 폴더를 C:\Linux\ 폴더에 통째로 복사한다.
    - 부팅 후 root 사용자로 접속한다.
- GRUB의 내용을 변경하고 부팅 화면 실행하기
    - vi 에디터(Visual editor)로 /etc/default/grub 파일을 열어 다음과 같이 세 행을 변경하고 저장한 후 닫는다.
    - 변경한 내용을 적용하기 위해 update-grup 명령을 입력한다.
    ```
    GRUB_TIMEOUT_STYLE=countdown    - 부팅 시 대기 시간을 보여줌
    GRUB_TIMEOUT=20    - 부팅 대기 시간을 20초로 변경
    GRUB_DISTRUPTOR=“Cookbook Linux”    - 초기 화면의 글자 변경
    ```
    - reboot 명령으로 재부팅하면 20초 동안 카운트하는 것이 보인다. Esc를 누르면 글자가 바뀐 GRUB 초기 화면이 나타난다.
    - 첫 행에서 Enter를 누르고 root 사용자로 접속한다.
- GRUB 전용 사용자와 비밀번호 생성하기
    - vi 에디터로 /etc/grub.d/00_header 파일을 열어 마지막에 다음 네 행을 추가하고 저장한 후 닫는다. 이때 GRUB 사용자는 기존의 리눅스 사용자와 관련이 없으며 새로 지정하면 된다.
    ```
    cat << EOF
    set superusers=“grubuser”    --grubuser 사용자는 새로운 GRUB 사용자의 이름
    password grubuser 1234    -- ‘GRUB사용자 새비밀번호’ 형식으로 비밀번호 설정
    EOF
    ```
    - 변경한 내용의 적용을 위해 update-grub 명령어를 입력한다.
    - reboot 명령으로 시스템을 재부팅한다. 이제 GRUB을 편집하려면 앞에서 설정한사용자와 비밀번호가 있어야 한다. 편집하기 위해 Esc를 누르고 GRUB 화면으로 들어가서 E를 누르면 사용자와 비밀번호를 입력하는 창이 나타난다. 앞에서 설정한 사용자와 비밀번호를 정확히 입력해야 편집 모드로 들어갈 수 있다.
    - 편집모드에서는 설정을 변경하지 말고 Ctrl+X를 눌러서 부팅한다.
