# <font color="#336699">셸 스크립트 프로그래밍</font>
## <font color="#003366">셸의 개념과 특징</font>
### <font color="#003366">셸의 개념</font>
리눅스의 셸은 명령과 프로그램을 실행할 때 사용하는 인터페이스로, 셸은 윈도우의 명령 프롬프트와 비슷해 보이지만 훨씬 강력한 프로그래밍 기능이 있다.<br>
셸은 사용자가 입력한 명령을 해석하여 커널에 전달하거나 커널의 처리 결과를 사용자에게 전달하는 역할을 한다. 셸은 Server(B)의 텍스트 모드처럼 명령을 입력하는 환경과 비슷하다. 우분투에서 기본적으로 사용하는 셸은 bash(Bourne Again Shell)이다. bash의 특징은 다음과 같다.<br>
- 에일리어스(alias, 명령단축) 가능
- 히스토리 기능(↑ 또는 ↓)
- 연산 기능
- job Control 기능
- 자동 이름 완성 기능(tab)
- 명령 편집 기능

### <font color="#003366">셸의 명령어 처리 방법</font>
셸 명령문에는 다양한 옵션과 인자(argument)를 사용한다. 셸 명령문의 형식은 다음과 같다.
```
(프롬프트) 명령 [옵션...] [인자...]
```
<br>다음은 옵션과 인자를 사용한 셸 명령문의 예이다.
```
#ls -l
#rm -rf /mydir
#find . / -name "*.conf"
```
<br>

### <font color="#003366">환경 변수</font>
셸에서는 여러가지 환경 변수 값을 불러올 수 있으며 설정된 환경 변수는 <b>echo $환경변수</b> 명령으로 확인한다. 예를 들어 호스트 이름을 출력하려면 <b>echo $HOSTNAME</b> 명령을 실핸한다. 다음 표에 bash 셸의 주요 환경 변수를 정리했다.<br><br>
![1](/images/2024-04-09-linux-report-Chapter11/Picture1.png)<br>
<b>export 환경변수=값</b> 명령을 실행하면 환경 변수 값을 변경할 수 있으며, 확인할 때는 <b>printenv</b> 명령을 실행한다.<br>

## <font color="#003366">셸 스크립트 프로그래밍 기본</font>
### <font color="#003366">셸 스크립트 작성</font>
리눅스의 셸 스크립트는 C언어와 유사한 방법으로 프로그래밍이 가능하다. 대부분의 리눅스는 C언어로 작성되어 있어 프로그래밍 언어를 다루어본 적이 있다면 셸 스크립트에 쉽게 익숙해질 수 있다. 이런 셸 스크립트는 주로 vi 에디터나 gedit으로 작성한다.<br>
리눅스의 많은 부분이 셸 스크립트로 작성이 되어있기 때문에 셸 스크립트를 알아두면 도움이 될 수 있다.<br>
예를 들어 GRUB의 설정 파일은 /boot/grub/grub.cfg에서도 셸 스크립트 문법을 사용한다.<br><br>

<b>vi name.sh</b> 또는 <b>gedit name.sh</b> 명령을 실행하여 다음과 같은 간단한 셸 스크립트를 작성해보자. 셸 스크립트 파일의 확장명은 *.sh를 사용하는 것이 좋다.
```
name.sh 파일

1   #!/bin/sh
2   echo "사용자 이름: " $USER
3   echo "홈 디렉터리: " $HOME
4   exit 0
```
<b>1행</b>: 
    첫 행에 반드시 써야 하며, 특별한 형태의 주석(#!)으로 bash 셸을 사용하겠다는 의미이다.<br>
<b>2행</b>:
    <b>echo</b>는 화면에 출력하는 명령어이다. 먼저 '사용자 이름:'이라는 글자를 출력하고 옆에 $USER라는 환경 변수의 내용을 출력한다.<br>
<b>4행</b>: 
    종료 코드를 반환한다.<br>

### <font color="#003366">셸 스크립트 실행</font>
크게 두 가지 방법이 있다.<br>

- sh 명령으로 실행: <b>sh 스크립트파일</b> 명령으로 실행하는 방법은 셸 스크립트 파일의 속성을 변경할 필요가 없다는 것이 장점이다.
- '실행 가능' 속성으로 변경 후 실행: 먼저 셸 스크립트의 파일 속성을 '실행 가능'으로 변경한 후 <b>./스크립트파일</b> 명령을 실행한다. './'는 현재 디렉터리를 의미하며, <b>chmod +x 파일명</b>은 현재 파일의 속성에 '실행 가능' 속성을 추가하라는 명령이다.<br>

### <font color="#003366">변수</font>
변수는 값을 계속 변경하여 저장하는 개념이다. 셸 스크립트의 구조는 변경할 필요가 없지만 설정 값이 상황에 따라 다를 때는 변수를 바꾸는 방식으로 프로그래밍하면 편리하다.<br>

- 변수의 개요<br>
    셸 스크립트에서는 변수를 사용하기 전에 미리 선언하지 않으며, 처음 변수에 값이 할당되면 자동으로 변수가 생성된다. 변수에 들어가는 모든 값은 문자열(String)로 취급되고 숫자를 넣어도 동일하게 취급된다. 변수명은 대문자와 소문자를 구분한다. 또한 변수를 대입할 때 '=' 앞뒤에 공백이 없어야 한다.<br>

- 변수의 입력과 출력<br>
    $가 포함된 글자를 출력하려면 ' '로 묶거나 앞에 \를 넣어야 한다.<br>

- 숫자 계산<br>
    변수 값을 연산하려면 expr 키워드를 사용한다. 단, 수식과 함께 키보드 1옆의 `(백쿼트)로 반드시 묶어야 한다. 또한 수식에 괄호를 사용하려면 그 앞에 반드시 \를 넣어야 한다. *도 예외적으로 앞에 \를 넣어야 한다.<br>
```
1   #!/bin/sh
2   num1=100
3   num2=$num1+200
4   echo $num2
5   num3=`expr $num1 + 200`
6   echo &num3
7   num4=`expr \(&num1 + 200\) / 10 \* 2`
8   echo $num4
9   exit 0
```
<br><br>
- 파라미터 변수<br>
    파라미터 변수는 $0, $1, $2와 같은 형태이며, 이는 실행하는 명령의 각 부분을 변수로 지정한다는 의미이다. 예를 들어 <b>apt-get -y install gftp</b> 명령의 경우 다음과 같이 파라미터 변수를 지정할 수 있다. 즉 $0에는 apt-get이, $1에는 -y가, $2에는 install이, $3에는 gftp가 저장된다. 명령어 전체의 마라미터 변수는 $*로 나타낸다.<br>

### <font color="#003366">if문과 case문</font>
- 기본 if문<br>
```
if [조건]
then
    참인 경우 실행
fi
```
위와 같은 형식을 가진다.<br>

- if~else문<br>
```
if [조건]
then
    참인 경우 실행
else
    거짓인 경우 실행
fi
```
위와 같은 형식을 가진다.<br>

## <font color="#003366">셸 스크립트 프로그래밍 응용</font>
### <font color="#003366">셸 스크립트 응용 기능</font>
- 사용자 정의 함수<br>
```
함수명 () {
    내용
}
함수명
```
위와 같은 형식을 가진다.<br>

- 함수의 파라미터 사용
```
함수명 () {
    $1, $2, ... 등을 사용
}
함수명 파라미터1 파라미터2 ...
```
위와 같은 형식을 가진다.<br>

- eval<br>
문자열을 명령문으로 인식해서 실행한다.<br>
```
1   #!/bin/sh
2   str="ls -l eval.sh"
3   echo $str
4   eval $str
5   exit 0
```
<b>3행</b>: str 변수의 값인 'ls -l eval.sh'라는 글자를 그대로 출력한다.
<b>4행</b>: str 변수의 값인 'ls -l eval.sh'를 명령으로 인식하여 실행한다.<br>

- export<br>
특정 변수를 전역 변수로 만들어 모든 셸에서 사용할 수 있다.<br>
```
exp1.sh
1   #!/bin/sh
2   str="ls -l eval.sh"
3   echo $var1
4   eval $var2
5   exit 0
```
```
exp2.sh
1   #!/bin/sh
2   var1="지역 변수"
3   export var2="외부 변수"
4   sh exp1.sh
5   exit 0
```
<b>exp1.sh 2~3행</b>: var1, var2 변수를 출력한다. var2는 exp2.sh에서 외부 변수로 선언했다.<br>
<b>exp2.sh 2행</b>: var1에 값을 넣는다. 일반 변수(지역 변수)이므로 현재 프로그램인 exp2.sh에서만 사용된다. 즉, exp1.sh의 var1과는 이름만 같을 뿐 다른 변수이다.<br>
<b>exp2.sh 3행</b>: var2를 외부 변수로 선언하고 값을 넣는다. 외부 프로그램(exp1.sh)에서도 var2를 사용할 수 있다.<br>
<b>exp2.sh 4행</b>: exp1.sh를 실행한다.<br>

- printf<br>
C언어의 printf() 함수와 비슷하게 형식을 지정하여 출력할 수 있다.
```
1   #!/bin/sh
2   var1=200*5
3   var2="재미있는 비기너 우분투~~"
4   printf "%5.2f \n\n \t %s \n" $var1 "$var2"
5   exit
```
<b>3행</b>: 공백이 있으므로 ""로 묶어야 한다.<br>
<b>4행</b>: %5.2f는 총 다섯 자리이며 소수점 아래 두 자리까지 출력하라는 의미이다. \n은 한 행을 넘기는 개행 문자이고, \t는 Tab 문자이며, %s는 문자열을 출력한다. &var2의 경우 값 중간에 공백이 있으므로 변수 이름을 ""로 묶어야 오류가 발생하지 않는다.<br>

- set과 $(명령)<br>
    리눅스 명령을 결과로 사용하려면 '$(명령)' 형식을 이용하고, 결과를 마라미터로 사용하려면 set 명령어를 이용한다.<br>

- shift<br>
    파라미터 변수를 왼쪽으로 한 단계씩 아래로 시프트(이동)한다.
